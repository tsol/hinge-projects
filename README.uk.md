## у чому суть

знаєш відчуття, коли приходить ідея і хочеться швидко накидати прототип просто з телефона лежачи на дивані? ну от.

**HINGE** — це крихітний Vite плагін, який вбудовує у твій фронтенд-проєкт оверлей. тикаєш на шестерню, пишеш що хочеш змінити, і AI-агент редагує код — HMR підхоплює і зміни одразу з'являються в браузері. без IDE, без термінала, навіть за стіл сідати не треба.

ти запускаєш **cloudflared тунель** до свого локального дев-сервера, отримуєш публічний URL, відкриваєш його на телефоні — і вайбкодиш звідки завгодно.

встановлення — одна команда. буквально **одна**:

```
https://raw.githubusercontent.com/tsol/hinge-projects/main/skills/hinge-projects-install.md
```

кидаєш посилання агенту. готово.

---

## що отримуєш

- дев-сервер з Hinge панеллю на будь-якій твоїй машині
- публічний URL — доступ з телефона, планшета, звідки завгодно
- **створення нових Vue/React проєктів** з уже вбудованим Hinge — одна команда
- агент знає структуру проєкту, читає/пише файли, все живе в браузері
- можеш навіть дати доступ нетехнічному другу — він зможе залишати задачі в черзі

## як користуватися з Telegram (або будь-якого чату)

коли все запущено, просто кажеш агенту:

| Ти кажеш | Агент відповідає |
|----------|------------------|
| `що працює?` | `hinge → https://xxx.trycloudflare.com ✅` + список проєктів |
| `який урл у krollo?` | `https://plasma-sun-jon-motors.trycloudflare.com` |
| `створи vue проєкт test-app` | `✅ Created test-app … url=https://...` |
| `створи react блог test-blog` | `✅ Created blog … url=https://...` |
| `видали test-app` | `✅ Removed test-app` |
| `перезапусти krollo` | `✅ Restarted, tunnel: https://...` |

агент всередині використовує `hip` — те ж саме що в терміналі:

```bash
hip                          # список проєктів + статус + URLs
hip hinge status             # детальний статус hinge
hip krollo start             # запуск дев + тунель
hip krollo restart           # перезапуск, тунель лишається
hip create vue test-app --start
hip create react blog --start
hip delete test-app --force
```

## команди

| Команда | Що робить |
|---------|------------|
| `hip` | Список проєктів зі статусом і посиланнями |
| `hip <name> start` | Запустити дев-сервер + тунель |
| `hip <name> stop` | Зупинити дев-сервер + тунель |
| `hip <name> restart` | Перезапустити; зберегти тунель |
| `hip <name> status` | Детальний статус |
| `hip create vue <name> [--start]` | Створити Vue 3 + TS + Hinge |
| `hip create react <name> [--start]` | Створити React + TS + Hinge |
| `hip create list` | Доступні фреймворки |
| `hip delete <name> --force` | Видалити проєкт |

## ⚠️ друже, серйозно

**hinge-projects створює публічні Cloudflare тунелі прямо в твій локальний дев-сервер.**

- будь-хто з URL може відкрити твою апку і Hinge панель
- **жодної аутентифікації** — немає логіна, пароля, токена
- це суворо для **особистого прототипування та вайбкодингу**
- **не** спрямовуй це на продакшен, бази даних, криптогаманці чи бабусин рецепт печива

я попередив. використовуй на свій страх і ризик.
